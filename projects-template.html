CampusRide E-Hailing App

A full-stack mobile application for university students, built from the ground up to provide a safe and affordable community-focused transportation solution.

The Challenge

University students often face high costs and limited availability with commercial ride-sharing services. The goal was to create a dedicated, affordable, and community-focused e-hailing platform for students within and around the campus area, aiming to replace the popular but discontinued "Student Grab" service.

My Solution

I developed a complete, full-stack e-hailing application from scratch. The front-end was built using Flutter, allowing for a single codebase for both Android and iOS. For the backend, I utilized Firebase for its real-time database capabilities, user authentication, and cloud functions, which enabled features like live driver tracking and secure data handling.

Key Features

  • Real-time GPS Driver Tracking: Integrated Google Maps API to show live vehicle movement for both passengers and drivers.
  • Secure Authentication & Profiles: Firebase Auth for secure email/password and social logins, with distinct profiles for drivers and riders.
  • In-App Ride Booking & Management: A seamless flow from setting pickup/destination to ride completion.
  • Fare Calculation: A simple algorithm based on distance and a base fare to provide transparent pricing.

Code Highlight

Here is a snippet of Dart code from the Flutter project, demonstrating the function to fetch a driver's live location from Firestore and update the map marker.


// Example Flutter (Dart) code
StreamSubscription? driverLocationSubscription;

void _getDriverLocation(String driverId) {
  driverLocationSubscription = FirebaseFirestore.instance
      .collection('drivers')
      .doc(driverId)
      .snapshots()
      .listen((snapshot) {
        if (snapshot.exists) {
          GeoPoint pos = snapshot.data()!['currentLocation'];
          LatLng driverLatLng = LatLng(pos.latitude, pos.longitude);
          _updateMarker(driverLatLng);
        }
      });
}

What I Learned

This project was an incredible learning experience in full-stack development and project management. The key takeaways were:

Managing real-time data streams with Firebase is powerful but requires careful state management on the client-side to prevent memory leaks and ensure UI consistency.

Additionally, I gained deep experience in integrating third-party APIs like Google Maps and learned the importance of designing a scalable database schema from the very beginning.
